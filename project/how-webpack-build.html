    # 打包结果运行原理初探

    我们先写一段简单的代码用来编译。

    // src/index.js
import test from './test'
window.test = test
// src/test.js
export default function test() {
  return 'test'
}

    将 Webpack 工作模式设置为 none，这样 Webpack 就会按照最原始的状态进行打包，所得到的结果更容易理解和阅读。

    按照 none 模式打包完成后，我们打开最终生成的 bundle.js 文件，如下图所示：

    我们将代码折叠起来， 可以发现整体代码是一个立即执行函数, 这个函数时webpack工作入口（webpackBootstrap），参数接受一个modules数组。
