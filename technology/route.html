    # 1.什么是前端路由

    传统的开发方式中, 用户访问一个url, 服务器根据访问的地址返回一个对应的HTML文件, 一个应用包含多个url, 多个url对应多个HTML文件, 我们称之为多页面应用; 因为页面路由的控制由服务器负责, 这种路由方式称为后端路由.

    由于每次切换页面都需要重新向服务器发送请求, 整个页面也需要重新加载, 对用户体验也有很大的影响, 所以出现了单页面应用.

    单页面应用其实也是由多个页面组成, 只是url的变化并不会引起整个页面的刷新, 也不会向服务器发送请求获取新的HTML文件, 页面内容的变化由前端来处理, 这种方式成为前端路由.

    # 2.实现前端路由的原理

    前端路由的原理就是通过url与渲染函数的映射关系, 把当前url对应的UI展示出来. 目前有两种实现方案,如下:

    # 2.1 hash路由

    url中#号后边的部分称为hash. 利用改变hash来改变url页面不会刷新的特性, 配合监听 hash 变化的事件hashchange, 我们可以很简单的完成一个基于hash的路由方法,如下:

      // html文件
  <div>
    <ul>
      <li>
        <a href="#/">aaaaa</a>
      </li>
      <li>
        <a href="#/b">bbbbb</a>
      </li>
      <li>
        <a href="#/c">ccccc</a>
      </li>
    </ul>
    <div id="app" />
  </div>
  // 实现路由方法
  class HashRouter {
    constructor() {
      this.routes = {}
      this.current = ''
    }
    addRoute(path, callback) {
      this.routes[path] = callback || function () {}
    }
    render() {
      this.current = location.hash.slice(1) || '/'
      const cb = this.routes[this.current]
      cb && cb()
    }
    init() {
      window.addEventListener('load', this.render.bind(this), false)
      window.addEventListener('hashchange', this.render.bind(this), false)
    }
  }
  // 调用
  const Router = new HashRouter()
  Router.init()
  Router.addRoute('/', () => {
    document.getElementById('app').innerHTML = 'aaa'
  })
  Router.addRoute('/b', () => {
    document.getElementById('app').innerHTML = 'bbb'
  })
  Router.addRoute('/c', () => {
    document.getElementById('app').innerHTML = 'ccc'
  })

    # 2.2 history路由

    HTML5规范中, History新增几个API:

    pushState:

    浏览器不会向服务端请求数据，直接改变url地址，可以类似的理解为变相版的hash；同时浏览器会记录pushState的历史记录，可以使用浏览器的前进、后退功能作用

    replaceState:

    不同于pushState，replaceState是替换历史记录

    同时新增了popstate事件, 用于监听前进或者后退按钮的点击(但是监听不到pushState和replaceState的调用).

    所以我们可以使用popstate事件来监听前进后退,使用pushState和replaceState封装一个跳转方法,来实现前端路由, 代码如下:

    // html文件
  <div>
    <ul>
      <li>
        <a href="/">aaaaa</a>
      </li>
      <li>
        <a href="/b">bbbbb</a>
      </li>
      <li>
        <a href="/c">ccccc</a>
      </li>
    </ul>
    <div id="app" />
  </div>
// 实现路由方法
  class HistoryRouter{
    constructor() {
      this.routes = {}
      this.current = ''
    }
    addRoute(path, callback) {
      this.routes[path] = callback || function () {}
    }
    observePopstate() {
      this.render()
    }
    render() {
      this.current = location.pathname
      const cb = this.routes[this.current]
      cb && cb()
    }
    init() {
      window.addEventListener('popstate', this.observePopstate.bind(this))
      // 拦截a标签的默认行为, 使用pushState修改url并手动触发UI更新
      const linkList = document.querySelectorAll('a[href]')
      linkList.forEach(el => el.addEventListener('click',  (e) => {
        e.preventDefault()
        history.pushState(null, '', el.getAttribute('href'))
        this.render()
      }))
    }
  }
  const Router = new HistoryRouter()
  Router.init()
  Router.addRoute('/', () => {
    document.getElementById('app').innerHTML = 'aaa'
  })
  Router.addRoute('/b', () => {
    document.getElementById('app').innerHTML = 'bbb'
  })
  Router.addRoute('/c', () => {
    document.getElementById('app').innerHTML = 'ccc'
  })

    # 3.两种实现方式的对比

    # hash路由的优缺点

    优点:

    • 兼容性好(IE8及以上)
    • 不需要服务端进行额外的配置
    • hash在重定向的时候可能有坑,如微信回调

    缺点:

    • 如果页面还需要锚点功能则会与路由冲突
    • 丑

    # history路由的优缺点

    优点:

    • 不用担心参数丢失
    • 好看
    • 可以使用history.state来获取当前url对应的状态信息

    缺点:

    • 兼容性相对差一些(IE10及以上)
    • 需要后端支持, 因为这个网址并不是对应一个真实的html文件, 直接访问路由会404, 后端需要返回根路径的HTML文件, 交由前端路由模块处理
